# Review Patterns — 2026-03-01

## ORC-47: Multi-Profile Frontend Profile Selector

### Verdict: REQUEST_CHANGES (4 blockers)

### Patterns observed

1. **React Compiler + optional chaining in deps**: Using `activeProfile?.id` in a `useCallback` dep array causes `react-hooks/preserve-manual-memoization`. Pattern fix: destructure to `const activeProfileId = activeProfile?.id` before the hook.

2. **setState-in-effect anti-pattern**: Synchronous `setState` inside `useEffect` for derived/synced state is flagged by React Compiler ESLint plugin. Prefer `useMemo` for values derived from other state/props.

3. **Type mismatch between frontend types and backend serialization**: Required fields in a shared interface (`agentCount`, `lastActivity`) not populated by the backend service — TypeScript misses it because backend and frontend compile separately. Watch for this in cross-boundary types.

4. **URL construction inconsistency**: Most hooks use `URLSearchParams` correctly; DashboardPage.tsx used string interpolation with ternaries. Flag inconsistency in code reviews.

5. **AbortSignal not propagated to interval callbacks**: Pattern: AbortController only covers the mount fetch, not periodic refreshes. Fix with a `mountedRef`.

### CI gate: `bun run lint` catches all 3 HIGH ESLint issues. Always run lint before approval.
